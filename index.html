Overview

DevDocs Pro is a documentation-as-code platform that auto-generates beautiful, searchable API docs from your OpenAPI spec. It handles hosting, versioning, search, and SDK generation â€” so you focus on building, not documenting.

Note Looking for the changelog? Check out What's new in v3.0 for full details on the latest features including semantic search and the new Bento layout engine.

How it works

Connect your spec, configure your project, and deploy. DevDocs Pro parses your OpenAPI / AsyncAPI file and generates a fully interactive documentation portal with live code playgrounds.

1

Connect your spec

Point to your openapi.json or openapi.yaml file â€” local path or remote URL. DevDocs Pro supports OpenAPI 3.0, 3.1, and AsyncAPI 2.x.

2

Configure your project

Set your project name, theme, custom domain, and access controls in devdocs.config.ts. Sane defaults mean you rarely need to touch it.

3

Deploy anywhere

Push to production with devdocs deploy. Supports Vercel, Netlify, Cloudflare Pages, or any Node.js host. CI/CD integration included.

Installation

Install the CLI globally and the SDK as a dependency:

npm install -g @devdocs/cli
npm install @devdocs/sdk
pnpm add -g @devdocs/cli
pnpm add @devdocs/sdk
yarn global add @devdocs/cli
yarn add @devdocs/sdk
bun add -g @devdocs/cli
bun add @devdocs/sdk

Authentication

All API requests require a Bearer token. Generate yours from the dashboard.

auth.ts
TypeScript 
import { DevDocs } from '@devdocs/sdk'

// Initialize with your API token
const client = new DevDocs({
  token: process.env.DEVDOCS_TOKEN,   // required
  version: 'v3',                       // default: latest
  timeout: 10_000,                    // ms, default: 30s
  retries: 3,                          // auto-retry on 5xx
})

Best practice Store your token in an environment variable. Never commit it to source control. Use .env.local locally and your CI/CD secret manager in production.

Your first API call

List your projects and fetch a specific page:

example.ts
TypeScript 
// List all projects
const projects = await client.projects.list({ limit: 10 })
console.log(projects.data) // Project[]

// Fetch a specific page
const page = await client.pages.get('proj_abc123', 'getting-started')
console.log(page.title)   // "Getting Started"
console.log(page.content)  // MDX string

// Search across all docs
const results = await client.search.query({
  q: 'authentication',
  limit: 5,
  projectId: 'proj_abc123'  // optional filter
})
results.data.forEach(r => console.log(r.title, r.score))

SDK Parameters

Parameter Type Required Description
token string Required Your API token from the dashboard
version string Optional API version. Defaults to "v3"
timeout number Optional Request timeout in ms. Default 30000
retries number Optional Auto-retry count on 5xx errors. Default 3
baseUrl string Optional Override the base URL (for self-hosted)

Rate limits The free plan is limited to 1,000 requests/day. Pro plan gets 100,000/day. Exceeding limits returns a 429 Too Many Requests response with a Retry-After header.
